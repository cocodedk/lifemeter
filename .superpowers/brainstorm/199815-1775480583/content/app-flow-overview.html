Issues found
🔴
No birthdate memory
App resets to today's date every launch. You explicitly asked for this to be fixed.
🔴
Date picker opens on today
For a birthdate, the default year should scroll to ~30 years ago, not 2026.
🟡
Mixed Danish + English
Card labels are English ("Days alive") but values are Danish ("Du har levet for: X Dage"). Inconsistent throughout.
🟡
Two-button confusion
"Change birth date" + "Continue" are two steps when one tap on the date should be enough to drive the flow.
🟡
Stat values embed labels
e.g., "Du har levet for: 12465 Dage" — label and number are in one string. Hard to style the number prominently.
🟡
TarotCardActivity is a stub
populateTable() is empty, count is always 0. Has a debug Toast("OnCreate") still visible to users.
🟢
Design system is solid
Color palette, cards, typography and buttons are already well-defined and consistent.
🟢
Live timer works
Session counters (deaths, births, time on screen) tick correctly every second.
First design question

What language should the app use?

Currently it's a mix — labels in English, values in Danish. Pick one direction.

A

All Danish

Every label, button, and string in Danish. "Dage i live", "Sekunder i live", "Mad spist", etc.

B

All English

"Days alive", "Seconds alive", "Food consumed" — consistent English throughout.
