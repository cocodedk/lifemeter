App States
FIRST LAUNCH
No saved date
→ Welcome + date picker prominent
→
DATE PICKED
DatePickerDialog appears → on confirm, save to SharedPreferences + immediately reveal dashboard with animation
RE-LAUNCH
Saved date found
→ Dashboard loads immediately, no tap
→
Date card still visible at top (compact). User taps "Change birth date" → DatePickerDialog → picks new date → saved + dashboard updates live.
SWIPE DOWN
Pull down on scroll view → opens DatePickerDialog
→
SwipeRefreshLayout wraps the NestedScrollView. Pulling down = change date. Removes need to scroll back to top. Immediate feedback.
BACKGROUND / FOREGROUND
App moves to background, comes back
→
Session timer resets on resume (onResume restarts timer). "Time on screen" = truly this session, not cumulative since ever.
